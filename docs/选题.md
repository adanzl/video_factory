## 1. 核心定位与筛选标准

专为 **B 站创作激励** + **AI 全自动生产** + **AI 自动化质检** 设计，与 `需求.md` 中「科普向、禁止高风险题材」保持一致。

每条候选标题须同时满足以下三要素：

| 维度 | 要求 | 与流水线的关系 |
| ---- | ---- | -------------- |
| **画面易生成** | 无复杂实景、无真人、无难建模物体；**科普插画 / 信息图解**可稳定出图 | 降低画面质检回退率 |
| **事实固定无争议** | 科学常识、生活原理、数码参数等；可建知识库校验 | 支撑文案事实校验环节 |
| **长尾流量稳定** | 不过时、不追热点；长期有搜索与播放 | 持续产生激励收益 |

**淘汰原则**：任一维度明显不达标，或总分低于阈值（见 [§5](#5-机器打分模型)），不进入生产队列。

## 2. 四大量产赛道

按 **优先级** 排序；选题生成与配额分配时优先填满高优先级赛道。

### 赛道 1：日常科学原理（最高优先级）

| 项 | 说明 |
| -- | ---- |
| **收益参考** | 千次播放约 2.0～2.6 元（全站偏高） |
| **AI 适配** | 极强；卡通实验室 / 微观动画画风统一，质检匹配度高 |
| **内容特征** | 生活常见现象 + 科学原理解析 |
| **典型题材** | 水、电、温度、气压、化学反应、光学、日常用品原理 |
| **禁止** | 高深物理、医学人体、生物病理 |

### 赛道 2：生活避坑实用常识（稳定长尾）

| 项 | 说明 |
| -- | ---- |
| **收益参考** | 千次播放约 1.1～1.7 元 |
| **AI 适配** | 极高；手机操作、家居 / 生活场景易生成 |
| **内容特征** | 普通人误区、隐形扣费、省电技巧、避坑方法 |
| **运营优势** | 收藏率高、完播稳、约三个月持续长尾播放 |

### 赛道 3：数码小白避坑（高净值粉）

| 项 | 说明 |
| -- | ---- |
| **收益参考** | 千次播放约 1.3～2.0 元 |
| **AI 适配** | 高；手机 / 电脑 / 路由器 / U 盘界面生成稳定 |
| **内容特征** | 参数解读、选购误区、清理缓存、硬件常识 |
| **限制** | 不做新品测评、不做实时行情、不做拆机 |

### 赛道 4：古代冷门生活史（低波动兜底）

| 项 | 说明 |
| -- | ---- |
| **收益参考** | 千次播放约 1.2～1.8 元 |
| **AI 适配** | 古风动画画风统一，出错率低、质检通过率高 |
| **内容特征** | 古人衣食住行、无现代科技的替代方案 |
| **运营优势** | 争议少、违规风险低、内容不过时 |

## 3. 标准化标题公式

机器批量生成标题时，统一使用下列模板，兼顾 **算法友好**、**画面可生成**、**质检通过率**。

| 模板 | 占比 | 结构 | 示例 |
| ---- | ---- | ---- | ---- |
| **误区反问式**（主力） | 70% | 【日常误区】+ 反问 + 「多数人不知道的真相」 | 不锈钢能被磁铁吸住，就代表买到次品了？ |
| **反差好奇式** | 20% | 同样是 XX，为什么 A 和 B 差距这么大？ | 同样是水，矿泉水保质期 2 年，自来水 3 天就变质？ |
| **实操避坑式** | 10% | XX 暗藏猫腻？教你正确辨别方法 | 千兆宽带网速拉胯，多半是网线规格选错了？ |

**生成注意**：

- 标题需能映射到 [§2](#2-四大量产赛道) 中某一赛道，并带明确「误区 / 反差 / 避坑」钩子。
- 避免模糊指代（如「这个东西」「很多人」），便于分镜与 Prompt 拆分。
- 与 `需求.md` §4.1 一致：合规性、科普适配度、爆款潜力、搜索热度由多模型交叉打分，本文件 §5 为选题侧细化权重。

## 4. AI 生产约束

以下规则为 **硬性约束**；违反则选题或成片直接淘汰 / 回退。

| 类别 | 规则 |
| ---- | ---- |
| **画面** | 无真人出镜，全程动画；避免人物畸形类 AI 错误 |
| **场景** | 无动态复杂场景（车流、人群、高速运动等） |
| **表述** | 无主观观点：只讲确定性常识，不评价、不预测、不科普争议内容 |
| **时效** | 无时效性：不做热点、新闻、节日向内容 |
| **高危领域** | 禁止医疗、养生、理财、股市、时政、情感等（与 `需求.md` 内容边界一致） |

## 5. 机器打分模型

供代码与多模型 **自动打分** 使用；与 `需求.md` §4.1 阈值配置联动（默认 **0–100 分，低于 65 分废弃**）。

### 5.1 维度与权重

| 维度 | 权重 | 高分特征 | 低分 / 淘汰信号 |
| ---- | ---- | -------- | --------------- |
| **画面可生成度** | 30 | 单场景、卡通/示意、无真人无复杂运动 | 真人、实景、人群车流、难建模物体 |
| **事实确定性** | 30 | 有唯一标准答案，可知识库校验 | 争议话题、预测、主观评价、无法核验 |
| **用户好奇心** | 20 | 反常识、误区、悬念强 | 平淡陈述、无钩子、标题与正文无反差 |
| **合规与长尾** | 20 | 非高危、非时效、符合赛道边界 | 触达 §4 禁止项、蹭热点、易过时表述 |

### 5.2 与需求文档的对应关系

| 本文件维度 | `需求.md` §4.1 维度 |
| ---------- | ------------------- |
| 事实确定性 + 合规与长尾 | 合规性 |
| 画面可生成度 + 赛道匹配 | 科普适配度 |
| 用户好奇心 + 标题公式匹配 | 爆款潜力 |
| 长尾、非时效、搜索向表述 | 搜索热度 |

### 5.3 实现建议（供代码）

```text
总分 = 画面可生成度×0.3 + 事实确定性×0.3 + 用户好奇心×0.2 + 合规与长尾×0.2
若 总分 < 65 或 任一硬性约束命中 → status = rejected
若 总分 >= 65 → status = queued（进入文案生产）
```

阈值、赛道配额、模板占比建议放入配置项，支持播放/收益回流后调参（见 `需求.md` §9 数据迭代）。
