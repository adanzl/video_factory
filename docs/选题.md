# 选题策略

> **定位**：面向 [B 站 AI 全自动科普视频量产系统](./需求.md) 的标题入库与筛选规则。  
> **目标**：批量产出「易生成、可校验、长尾稳定」的科普向标题，供规则打分后进入生产队列。  
> **代码**：`backend/app/services/topic/`（配置见 `catalog.py`）。

## 1. 核心定位

每条候选标题须同时满足：

| 维度 | 要求 |
| --- | --- |
| **画面易生成** | 无真人、无复杂实景；科普插画 / 信息图解可稳定出图 |
| **事实固定** | 科学常识、生活规则等可核验内容 |
| **长尾稳定** | 不写具体时效、人名、赛果 |

**淘汰**：总分低于 **85** 或命中硬性约束 → `rejected`。

---

## 2. 三大分类

入库字段：`title.category`（原 `track` 已迁移）。

| 分类 | 说明 | 默认主题 |
| --- | --- | --- |
| **历史悬案** | 代号 + 悬念，知名人物/事件 | 中国历史悬案 |
| **科学原理** | 日常科普、误区辟谣、画面锚点 | 日常科学冷知识 |
| **时事相关科普** | 从热议抽长尾原理，剥离时效 | （关键词驱动） |

旧赛道值（如 `日常科学原理`）迁移时映射为 `科学原理`。

## 3. 标题模板

模板控制**标题句式**，由 `catalog.py` 按权重自动分配，用户无需手选。

### 历史悬案

| 模板 | 权重 | 结构 |
| --- | --- | --- |
| 悬念钩子式 | 35 | 历史代号 + 一句话悬念 |
| 未解之谜式 | 30 | 失踪 / 暴毙 / 反常画面 |
| 误区反问式 | 20 | 颠覆常识的历史细节 |
| 反差好奇式 | 15 | 同一人物 / 事件两面反差 |

### 科学原理

| 模板 | 权重 | 结构 |
| --- | --- | --- |
| 误区反问式 | 50 | 误区 + 反问 + 一步反驳 |
| 反差好奇式 | 30 | 同样是 XX，为何结果不同 |
| 实操避坑式 | 20 | 猫腻 + 辨别方法 |

### 时事相关科普

| 模板 | 权重 | 结构 |
| --- | --- | --- |
| 误区反问式 | 60 | 剥离时效后讲清原理 / 误区 |
| 反差好奇式 | 40 | 公众认知 vs 科学真相 |

## 4. 生成参数

API `POST /v_factory/api/topic/gen`：

| 参数 | 必填 | 说明 |
| --- | --- | --- |
| `category` | 是* | 三大分类之一 |
| `theme` | 二选一 | 主题方向 |
| `keywords` | 二选一 | 逗号分隔关键词 |
| `count` | 否 | 1–20，默认 10 |
| `save` | 否 | 是否入库并打分 |

\* 使用自定义 `system_prompt` / `user_prompt` 时可省略 `category`。

## 5. 机器打分

实现：`topic/scorers/`（按分类拆分），对外接口 `topic_mgr.score_title`。

```text
总分 = 画面×0.3 + 事实×0.3 + 好奇×0.2 + 合规×0.2
总分 < 85 → rejected
总分 ≥ 85 → queued
```

| 维度 | 权重 | 高分特征 |
| --- | --- | --- |
| 画面可生成度 | 30 | 具象名词、示意动画友好 |
| 事实确定性 | 30 | 可核验常识、误区辟谣 |
| 用户好奇心 | 20 | 反问、反差、悬念 |
| 合规与长尾 | 20 | 非高危、非时效标题 |

## 6. 代码结构

```text
topic/
  catalog.py       # 分类 + 模板权重
  topic_mgr.py     # 业务入口 + score_title
  prompts/         # 通用 / 格式 / builder
  parsers.py       # LLM JSON 解析
  text.py          # 标题截断
  scorers/         # 分类打分实现
```

`apply_title_schema` 仅建表；旧库 `track` 列须先跑一次性迁移（已完成则可忽略）。
