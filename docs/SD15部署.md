## 2. 实测可用版本栈

> 机器：**leo@mini** · AMD 7940HS + 780M（**gfx1103**）· 2026-06-24 逐项试通。

### 2.1 推荐组合（GPU 出图）

| 层级 | 组件 | 版本 |
| --- | --- | --- |
| 系统 | Ubuntu | **24.04.4 LTS** |
| 内核 | linux-oem-24.04c | **6.17.0-1025-oem** |
| GRUB | `amdgpu.cwsr_enable=0` | 必开（防 780M 采样崩） |
| 驱动/ROCm | hip-runtime-amd 等 | **7.2.26015**（`/opt/rocm-7.2.0`） |
| Python | WebUI venv | **3.10.20** |
| WebUI | AUTOMATIC1111 | **v1.10.1**（`82a973c`） |
| PyTorch | torch | **2.12.0+rocm7.14.0a20260624** |
| PyTorch | torchvision | **0.27.0+rocm7.14.0a20260624** |
| PyTorch | triton | **3.7.0+rocm7.14** |
| HIP/ROCm（venv） | | hip **7.14.60850** / rocm **7.14.0** |

**验证结果**（同一栈）：

- GPU `matmul` ✅  
- MIOpen `conv2d`（开/关/全开）✅  
- WebUI 512×512 txt2img ✅（无需 `--use-cpu sd`）  
- 日志：`Applying attention optimization: sdp`

### 2.2 系统层安装要点

```bash
# OEM 内核（ROCm 官方要求 Ryzen APU 使用 oem 内核）
sudo apt install -y linux-oem-24.04c
# /etc/default/grub 的 GRUB_CMDLINE_LINUX_DEFAULT 加：
#   amdgpu.cwsr_enable=0
sudo update-grub && sudo reboot

# ROCm 7.2 用户态（示例，已装可跳过）
sudo amdgpu-install -y --usecase=rocm --no-dkms
sudo usermod -aG render,video "$USER"
```

### 2.3 PyTorch 安装（gfx1103 专用）

WebUI 首次 `webui.sh --exit` 会装 **pytorch.org 默认 ROCm wheel**，
对 gfx1103 **不可用**（见 §2.4），装完后必须换掉。

#### 为什么不用「稳定版」PyTorch？

| 来源 | 版本示例 | mini 实测 |
| --- | --- | --- |
| pytorch.org `rocm6.4` | torch 2.8.0+rocm6.4 | GPU `invalid device function` |
| repo.radeon.com 7.2 正式 | torch 2.8/2.9 + rocm7.2 | gfx1103 **segfault** |
| TheRock nightlies | torch 2.12+rocm7.14 + device-gfx1103 | ✅ GPU 出图 |

结论：**系统 ROCm 可用 7.2 稳定包**（§2.1），但 **PyTorch 稳定渠道尚未带齐
gfx1103 的 rocBLAS/MIOpen 设备库**，目前只能走 nightlies 的
`torch[device-gfx1103]`（会附带 `amd-torch-device-gfx1103`、`rocm-sdk-*`）。

mini 当前 venv 实测包名：

```text
torch                  2.12.0+rocm7.14.0a20260624
amd-torch-device-gfx1103
rocm-sdk-device-gfx1103
```

等 AMD 在 pytorch.org 或 repo.radeon.com 发布**含 gfx1103 的稳定 wheel**
后，再改回稳定源；届时更新本节。

#### 安装命令（与 mini 一致）

```bash
SD_HOME=/mnt/data/stable-diffusion/webui
source "$SD_HOME/venv/bin/activate"
pip uninstall -y torch torchvision torchaudio \
  amd-torch-device-* rocm-sdk-* rocm 2>/dev/null || true
pip install --index-url https://rocm.nightlies.amd.com/whl-multi-arch/ \
  "torch[device-gfx1103]" "torchvision[device-gfx1103]"
python -c "import torch; print(torch.__version__); print(torch.cuda.is_available())"
deactivate
```

### 2.4 已知不可用组合（勿用）

| 组合 | 实测现象 |
| --- | --- |
| **torch 2.8 + rocm6.4** | `invalid device function`；仅 CPU 可出图 |
| **repo.radeon.com ROCm 7.2 wheel** | gfx1103 上 **segfault** |
| **系统 ROCm 6.4 + 通用 wheel** | 缺 `gfx1103` rocBLAS/Tensile |
| **Python 3.12** venv | CLIP / torch 安装易失败 |
| **内核 6.8 generic**（未换 oem） | ROCm 对 Ryzen APU 支持差 |

采样仍崩时，在 `webui-user.sh` 末尾加 `--use-cpu sd`（慢但稳）。

## 4. 一键部署

```bash
export REPO_ROOT=/mnt/data/project/video_factory   # 本仓库路径
export SD_HOME=/mnt/data/stable-diffusion/webui
export MODELS_DIR=/mnt/data/stable-diffusion/models

bash "$REPO_ROOT/deploy/sd15/deploy.sh"
```

`deploy.sh` 会：克隆 WebUI、安装 `webui-user.sh`、建 `models` 软链、
首次 `webui.sh --exit` 装依赖、调用 `sd-webui-ctl.sh start`。

### webui-user.sh 说明

模板：`deploy/sd15/webui-user.example.sh`

| 参数 | 作用 |
| --- | --- |
| `--lowvram` | 780M 核显必开 |
| `--no-half` / `--no-half-vae` | 减轻 fp16 采样问题 |
| `--opt-sdp-attention` | ROCm 推荐注意力 |
| `--nowebui` / `--api` / `--listen` | 仅 API |

采样仍崩时，见 §2.4，在 `COMMANDLINE_ARGS` 末尾加 `--use-cpu sd`。

### 首次安装常见问题

**CLIP `pkg_resources`**：

```bash
source "$SD_HOME/venv/bin/activate"
pip install -U pip setuptools wheel
pip install --no-build-isolation \
  https://github.com/openai/CLIP/archive/d50d76daa670286dd6cacf3bcd80b5e4823fc8e1.zip
deactivate
bash "$SD_HOME/webui.sh" --exit --skip-torch-cuda-test
```

**Git 克隆要密码**：在 `webui-user.sh` 设
`STABLE_DIFFUSION_REPO=https://github.com/w-e-w/stablediffusion.git`

## 6. 启停与验证

```bash
bash "$REPO_ROOT/deploy/sd15/sd-webui-ctl.sh" status
bash "$REPO_ROOT/deploy/sd15/sd-webui-ctl.sh" restart
bash "$REPO_ROOT/deploy/sd15/sd-webui-ctl.sh" logs
```

冒烟（512×512）：

```bash
curl -s -X POST http://127.0.0.1:7860/sdapi/v1/txt2img \
  -H "Content-Type: application/json" \
  -d '{"prompt":"red apple","steps":10,"width":512,"height":512,
"sampler_name":"Euler"}' \
  -o /tmp/smoke.json && wc -c /tmp/smoke.json
```

### systemd（可选）

```bash
sudo tee /etc/systemd/system/sd-webui.service << EOF
[Unit]
Description=SD WebUI API
After=network.target
[Service]
Type=simple
User=$USER
WorkingDirectory=$SD_HOME
ExecStart=$SD_HOME/webui.sh
Restart=on-failure
[Install]
WantedBy=multi-user.target
EOF
sudo systemctl enable --now sd-webui
```

## 8. 显存规则

1. 单次仅 1 个 LoRA  
2. LoRA ≤ 150MB（上限 200MB）  
3. 仅 2 底模 + ≤8 LoRA，禁 SDXL/FLUX  
4. 禁 ControlNet、高清修复  
5. 分辨率 ≤768，batch_size=1，批量串行

## 9. 故障排查

| 现象 | 处理 |
| --- | --- |
| CLIP / `pkg_resources` | §3 手动装 CLIP |
| Git 要用户名密码 | `STABLE_DIFFUSION_REPO` 镜像 |
| 模型列表空 | 查软链 `ls -la webui/models` |
| `hipErrorLaunchFailure` | §2.4；`amdgpu.cwsr_enable=0`；或 `--use-cpu sd` |
| VAEApprox 下载超时 | 预下 `models/VAE-approx/model.pt` |

日志：`~/sd-webui.log` 或 `sd-webui-ctl.sh logs`
