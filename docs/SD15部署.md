# SD1.5 · 780M 显存 · Linux 无 UI 部署指南

> **适配硬件**：780M 显存（AMD 7940HS 核显）+ 32G 内存 + Linux 无桌面  
> **业务场景**：生活经验实拍 + 科普插画，双管线 API 切换  
> **核心原则**：仅 SD1.5 底模 · 单 LoRA · 禁 ControlNet/高清修复 · 批量出图不 OOM  
> **配套脚本**：仓库 `deploy/sd15/` 目录（一键部署 + 双业务出图）

## 1. 方案概览

| 项目 | 配置 |
| --- | --- |
| 底模 A | **Realistic Vision V5.1** — 生活实拍（美食/家居/静物/人像） |
| 底模 B | **ToonYou Beta 6** — 科普插画（白底线稿/示意图） |
| LoRA | 生活组 4 个 + 科普组 4 个，单次仅启用 1 个 |
| 启动参数 | `--medvram --lowvram --nowebui --api`（官方 A1111，非 amdgpu fork） |
| API 端口 | 7860（A1111 默认） |
| 分辨率上限 | 768px（生活 768×576 横版，科普 576×768 竖版） |

**显存说明**：780M 核显远低于 SD1.5 常规需求（~4G+）。本方案通过 `--medvram`、32G 内存 + 16G Swap、单 LoRA、禁高耗功能，将部分计算卸载到内存/Swap。**首张图可能需 2～5 分钟**，属正常现象。

## 2. 前置检查

在服务器上依次执行：

```bash
# 确认系统与内存
uname -a
free -h

# 确认 AMD 核显可见
lspci | grep -i vga
# 或
lspci | grep -i amd

# 确认磁盘空间 ≥ 30G（2 底模 + 8 LoRA + WebUI）
df -h ~

# 确认 Python 3.10+
python3 --version

# 确认 ROCm GPU Agent（应出现 Vendor Name: AMD + KERNEL_DISPATCH）
rocminfo | grep -A5 "Marketing Name"
```

**推荐系统**：Ubuntu 22.04 / 24.04 LTS（ROCm 支持较好）。其他发行版需自行对照 ROCm 官方兼容列表。

## 3. 系统准备（Swap + 依赖）

### 3.1 创建 16G Swap（防批量 OOM）

> 若 `swapon --show` 已有 ≥16G Swap，跳过本节。

```bash
sudo fallocate -l 16G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile

# 永久生效
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab

# 降低 swap 使用倾向，优先用内存
echo 'vm.swappiness=10' | sudo tee /etc/sysctl.d/99-swappiness.conf
sudo sysctl -p /etc/sysctl.d/99-swappiness.conf

free -h   # 应看到 Swap 约 16G
```

### 3.2 安装基础依赖

```bash
sudo apt update
sudo apt install -y \
  git wget curl unzip \
  python3 python3-venv python3-pip \
  libgl1 libglib2.0-0 libsm6 libxext6 libxrender1 \
  ffmpeg
```

## 4. 安装 ROCm（AMD 核显）

> 若已安装 ROCm 且 `rocminfo` 正常，跳过本节。

```bash
# Ubuntu 22.04 示例（24.04 请查 https://rocm.docs.amd.com 最新文档）
wget https://repo.radeon.com/amdgpu-install/latest/ubuntu/jammy/amdgpu-install_6.3.60300-1_all.deb
sudo apt install -y ./amdgpu-install_6.3.60300-1_all.deb
sudo amdgpu-install -y --usecase=rocm

# 将当前用户加入 render/video 组
sudo usermod -aG render,video "$USER"
# 重新登录 SSH 会话使组生效

# 验证
rocminfo | head -30
```

若 `rocminfo` 报错，WebUI 会回退 CPU 模式（极慢但可用）。优先排查驱动后再继续。

## 5. 安装 Stable Diffusion WebUI

```bash
# 安装目录（可按需修改，建议选空间充裕的数据盘）
export SD_HOME="$HOME/stable-diffusion-webui"
git clone https://github.com/AUTOMATIC1111/stable-diffusion-webui.git "$SD_HOME"
cd "$SD_HOME"
```

**首次安装依赖**（只需执行一次，耗时 10～30 分钟）：

```bash
cd "$SD_HOME"
# 仅安装 Python 依赖，不启动 UI
bash webui.sh --exit --skip-torch-cuda-test
```

若 ROCm 环境下 torch 安装失败，手动指定 ROCm 版 PyTorch（以官方当前版本为准）：

```bash
pip install torch torchvision --index-url https://download.pytorch.org/whl/rocm6.2
```

## 6. 配置启动参数

编辑 `$SD_HOME/webui-user.sh`：

```bash
cat > "$SD_HOME/webui-user.sh" << 'EOF'
#!/bin/bash
export HF_ENDPOINT=https://hf-mirror.com
export COMMANDLINE_ARGS="--skip-torch-cuda-test --medvram --lowvram --nowebui --api --listen --port 7860 --opt-split-attention --disable-safe-unpickle"
EOF
chmod +x "$SD_HOME/webui-user.sh"
```

| 参数 | 作用 |
| ----------------------- | ----------------- |
| `--skip-torch-cuda-test` | 跳过 CUDA 检测（AMD/无 NVIDIA 必开） |
| `--medvram` | 中等显存优化，模型分块加载 |
| `--lowvram` | 低显存模式，780M 核显必开 |
| `--nowebui` | 仅启动 API，不打开网页 UI（勿写 `--no-webui`） |
| `--api` | 开放 REST API |
| `--listen` | 监听 0.0.0.0，允许远程调用 |
| `--opt-split-attention` | 拆分注意力计算，省显存 |

> **AMD 说明**：官方 [AUTOMATIC1111/stable-diffusion-webui](https://github.com/AUTOMATIC1111/stable-diffusion-webui) **没有** `--use-rocm` 参数；ROCm 靠 venv 里安装的 ROCm 版 PyTorch 自动生效。若需 `--use-rocm`，应换 [stable-diffusion-webui-amdgpu](https://github.com/lshqqytiger/stable-diffusion-webui-amdgpu) fork。

## 9. LoRA INT4 量化（可选，推荐）

量化后 LoRA 显存占用约减半，适合 780M 连续批量出图。

```bash
pip install bitsandbytes accelerate safetensors

# 使用仓库脚本批量量化（需先 clone 本仓库或复制脚本）
python deploy/sd15/quantize_lora.py \
  --input-dir "$SD_HOME/models/Lora" \
  --output-dir "$SD_HOME/models/Lora_int4"
```

量化完成后，在 API 脚本中把 LoRA 路径指向 `Lora_int4/`，或在生成前替换原目录：

```bash
mv "$SD_HOME/models/Lora" "$SD_HOME/models/Lora_fp16"
mv "$SD_HOME/models/Lora_int4" "$SD_HOME/models/Lora"
```

> A1111 原生不识别 INT4 LoRA。若量化后无法加载，**跳过本节**，继续使用 fp16 LoRA + `--medvram --lowvram`。

## 11. systemd 开机自启

```bash
sudo tee /etc/systemd/system/sd-webui.service << EOF
[Unit]
Description=Stable Diffusion WebUI API
After=network.target

[Service]
Type=simple
User=$USER
WorkingDirectory=$SD_HOME
Environment=SD_HOME=$SD_HOME
ExecStart=$SD_HOME/webui.sh
Restart=on-failure
RestartSec=30

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable sd-webui
sudo systemctl start sd-webui
sudo systemctl status sd-webui
```

## 13. 双业务 API 调用

仓库提供 Python 脚本，自动切换底模 + LoRA + 固定参数：

```bash
# 环境变量
export SD_API_URL="http://127.0.0.1:7860"
export SD_OUTPUT_DIR="$HOME/sd-output"

# 生活实拍
python deploy/sd15/generate.py \
  --business life \
  --lora Food_Photo \
  --prompt "home cooking, natural window light, clean table, depth of field" \
  --output cover_001.png

# 科普插图
python deploy/sd15/generate.py \
  --business science \
  --lora Textbook_Line_Art \
  --prompt "cell structure diagram, labeled parts, educational" \
  --output diagram_001.png

# 批量（串行，禁止并发）
python deploy/sd15/generate.py --batch tasks.json
```

`tasks.json` 格式：

```json
[
  {"business": "life", "lora": "Food_Photo", "prompt": "...", "output": "food_01.png"},
  {"business": "science", "lora": "Simple_Diagram", "prompt": "...", "output": "diag_01.png"}
]
```

### 固定生成参数（脚本内置，勿改）

| 参数 | 生活实拍 | 科普插图 |
| --- | --- | --- |
| steps | 20 | 22 |
| cfg_scale | 7 | 7 |
| 尺寸 | 768×576 | 576×768 |
| sampler | DPM++ 2M Karras | DPM++ 2M Karras |
| LoRA 权重 | 0.6～0.7 | 0.6～0.7 |
| 高清修复 | 关 | 关 |
| ControlNet | 关 | 关 |

## 14. 显存管控硬性规则

1. **同一轮绘图仅 1 个 LoRA**，禁止 `<lora:A:0.6> <lora:B:0.5>` 叠加
2. **拒绝 > 200MB 的 LoRA**，优先 ≤ 150MB
3. **仅保留 2 个底模 + ≤ 8 个 LoRA**，不存 SDXL/FLUX/巨型画风模型
4. **禁用** ControlNet、高清修复（Hires.fix）、多 ControlNet
5. **分辨率 ≤ 768px**，禁止 1024+ 或 batch_size > 1
6. **批量任务串行执行**，脚本内禁止 asyncio 并发 txt2img
7. OOM 时追加 `--lowvram`，仍失败则降分辨率至 512×384

## 15. 故障排查

| 现象 | 处理 |
| --- | --- |
| `CUDA out of memory` / ROCm OOM | 确认 `--medvram --lowvram`；减分辨率；关其他占 GPU 进程 |
| 启动极慢 / 首张图 > 10 分钟 | 780M 核显正常；检查 Swap 是否生效 `free -h` |
| LoRA 不生效 | 检查文件名与 `<lora:Name:weight>` 是否完全一致（区分大小写） |
| API 403 / 连接拒绝 | 确认 `--listen`；防火墙放行 7860 `sudo ufw allow 7860` |
| 模型列表为空 | 确认 `.safetensors` 在 `models/Stable-diffusion/`，重启 WebUI |
| Civitai 下载 403 | 删除 URL 中 `?token=`；或浏览器登录后重新复制直链 |
| `rocminfo` 失败 | 重装 ROCm；或确认 venv 内为 ROCm 版 PyTorch（官方 A1111 无 `--use-rocm` 参数） |

**日志位置**：

```bash
tail -f ~/sd-webui.log
# systemd 方式
journalctl -u sd-webui -f
```
