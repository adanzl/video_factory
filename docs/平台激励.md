# 主流短视频平台 · 创作激励统计

> **用途**：为本项目（9:16 竖屏、约 4:20–5:20 科普成片）选型分发与算账参考。  
> **口径**：政策变动频繁，**千播单价均为社区/媒体估算或本项目中性假设**，非官方承诺；开通条件以各平台创作者中心为准。  
> **关联**：[成本.md](./成本.md) §5（B 站盈亏）、[需求.md](./需求.md) §11.5（MVP 多平台）、[§4.8](./需求.md#48-多平台分发)（多平台分发）。

## 1. 按 API 投稿能力分类（主视图）

> **分类标准**：能否在 `publish` 阶段接入 Worker **稳定、可 OAuth、可批量**上传视频（含元数据/封面），而非偶发脚本或 MCN 代投。  
> **本项目成片**：9:16 竖屏 MP4，约 4:20–5:20，科普向。

### 1.4 风险分析（按 API 分组）

#### A. ✅ 可 API 组 — 风险画像

| 风险类型 | 等级 | 说明与应对 |
| --- | --- | --- |
| **激励政策变动** | 🔴 高 | 抖音伙伴计划替代中视频、B 站基础激励月 cap 2000 元等；**勿用单一千播假设指导选题**，仅 B 站数据进 [需求.md §9](./需求.md#9-数据迭代) 默认调参。 |
| **开通门槛 / 空转** | 🔴 高 | 抖音常见 **有效粉丝 ≥5 万** 才享播放收益；新号 API 能发但**可能零播放收益**；上线前 App 内确认权益。 |
| **多平台同片** | 🟠 中高 | 各平台对搬运、同质化、AI 标识要求不一；建议**标题/封面/简介差异化**，避免机械镜像触发限流或激励剔除。 |
| **API 与审核** | 🟠 中 | OAuth 过期、频控、类目不符会导致投递失败或过审慢；Worker 需重试 + 人工告警队列（见 [需求.md §10](./需求.md#10-人工干预边界)）。 |
| **规格错配** | 🟠 中 | 头条/西瓜偏横屏逻辑，9:16 竖屏收益是否等同**未验证**；百家号偏资讯搜索，科普 ROI 不确定。 |
| **技术对接成本** | 🟡 中低 | 字节系（抖音/头条/西瓜）部分链路可复用；快手需**单独开放平台**；百家号接口偏运营，开发量大于 B 站。 |
| **自动化收益** | 🟢 相对可控 | 一旦打通，边际分发成本趋近零；**B 站 ROI 可算账**，其余平台收益单独记账。 |

**本组策略**：B 站 P0 全自动化 + 回流闭环；抖音/快手 P2 接 API 但**不纳入首月 15 条盈亏表**；头条/西瓜/百家号 P4 有余力再测。

---

#### B. ⚠️ 手传为主组 — 风险画像

| 风险类型 | 等级 | 说明与应对 |
| --- | --- | --- |
| **人力瓶颈** | 🔴 高 | 无法并入 Worker 无人值守链路；日批放大后**手传不可扩展**，易成为产能天花板。 |
| **流程遗漏** | 🔴 高 | 视频号须 **原创声明** 才计分成；支付宝须满足粉丝/条数/播放门槛；漏一步则**有播放无收益**。 |
| **收益测算失真** | 🟠 中高 | 支付宝「30 元/万播」等为媒体报道，垂类差异大；视频号依赖广告填充，**不宜写入盈亏公式**。 |
| **准入不确定** | 🟠 中 | 视频号部分账号需邀请/内测；小红书蒲公英仍在试点；开通状态需**按账号实测**。 |
| **合规边界** | 🟠 中 | 支付宝对理财/医疗类敏感，与本项目禁理财/医疗边界需对齐；生活/科普类相对安全。 |
| **二期 API 不确定** | 🟡 中 | 平台可能开放能力或收紧 MCN 通道；架构上保留 `ManualPublishAdapter` 接口，避免与 API 适配器耦合。 |
| **增量价值** | 🟢 存在 | 同片二次分发边际成本低（仅人力）；视频号社交推荐、支付宝有效播放单价传闻较高，**值得手传验证**。 |

**本组策略**：成片下载 + 检查清单手传（P1）；收益与粉丝门槛单独台账；蒲公英正式化后再评估是否升 P2 API。

---

#### C. ❌ 非 API 组 — 风险画像

| 风险类型 | 等级 | 说明 |
| --- | --- | --- |
| **赛道错配** | 🔴 高 | 短剧厂牌、合同制项目与批量科普流水线目标不一致，投入对接 ROI 极低。 |
| **弱平台空耗** | 🟠 中 | 微视等用户与激励规模弱，分散精力不划算。 |

**本组策略**：不纳入 MVP 分发与盈亏表；仅当商务机会出现再单独立项。

### 1.5 总览（按激励梯队，供算账参考）

| 梯队 | 平台 | 核心激励名称 | 变现模式 | 千播参考（估） | API 投稿 |
| --- | --- | --- | --- | --- | --- |
| **A · 首选** | **B 站** | 创作激励计划 + 广告分成等 | 综合质量/互动；基础激励月 cap 2000 元 | **约 1～4 元**（中性 **2.3**） | ✅ |
| **A** | **微信视频号** | 创作分成计划 | 评论区/主页流广告分成 | 差异大 | ⚠️ 手传 |
| **A** | **支付宝生活号** | 创作·分成计划 | 有效播放现金分成 | 社区约 **30 元/万播** | ⚠️ 手传 |
| **B · 重要** | **抖音** | 创作者伙伴计划 | 时长 + 质量综合收益 | 无稳定公开单价 | ✅ |
| **B** | **快手** | 光合 + 任务/分账 | 流量助推 + 商单/分账 | 非统一千播 | ✅ |
| **B** | **小红书** | 蒲公英分成 + 流量券 | 中插广告分成（试点） | 待上线 | ⚠️ 手传 |

| **C** | **微博 / 百家号** | 广告共享 / 内容联盟 | 曝光分成 | 无统一千播 | ⚠️ |
| **D · 专项** | 优酷 / 腾讯 / 短剧等 | 合同制分账 | 按项目 | 不适用 | ❌ |
| **E · 弱相关** | 微视、皮皮虾等 | 无稳定激励 | — | — | ❌ |

结论（本项目）

- **全自动 publish + [需求.md §9](./需求.md#9-数据迭代) 回流**：仅 **B 站**（P0）；抖音/快手接 API 但不进首月盈亏表（P2）。
- **同片增量、手传验证**：**视频号、支付宝**优先（P1）；小红书观察蒲公英规则（P3）。  
- **风险总原则**：API 组防「能发无收益」与政策变动；手传组防流程遗漏与人力瓶颈；**同一 MP4 多平台时仅 B 站数据指导调参**。

## 3. 与本项目 MVP 的对应关系（API 能力 × 优先级）

| ✅ 可 API | **P4** | 头条、西瓜、百家号 | 有余力再测；竖屏/横屏与收益口径需实测 |
| ⚠️ 手传为主 | **P1** | 视频号、支付宝 | 成片下载 + 原创声明/门槛检查清单手传；收益单独记账 |
| ⚠️ 手传为主 | **P3** | 小红书、微博 | 观察蒲公英正式规则；微博科普 ROI 低，低优先 |
| ❌ 非 API | — | 短剧厂牌、弱平台 | 不纳入 MVP |

**重要**：同一 MP4 多平台分发时，**仅 B 站数据**用于 [需求.md §9](./需求.md#9-数据迭代) 默认调参；其他平台收益单独记账，避免用未验证的千播假设指导选题。手传平台漏做「原创声明」等步骤会导致有播放无收益（见 §1.4 B 组）。

## 4. 不含或弱「创作激励」的说明

以下**不算**本项目意义上的「主流短视频创作激励」主战场，故不展开：

- **纯直播打赏/带货**（各平台均有，与成片流水线无关）  
- **短剧厂牌分账**（优酷、红果、快手磁力新剧等）——合同制，非批量科普  
- **微视、皮皮虾、好看视频**等——用户与激励规模弱  
- **海外 TikTok Creator Rewards**——需独立账号与合规，不在国内方案内
