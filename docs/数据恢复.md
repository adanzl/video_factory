# 数据恢复方案

## 背景

`prepare_for_action("script")` 清空了 job 29 的 script_json、video_segment 表、所有媒体文件及数据库路径。

## 恢复清单

| 项目 | 恢复方式 | 状态 |
|------|---------|------|
| script_json | 从 job_log 表提取（id=2887，81KB） | ✅ |
| video_segment（31条） | 从 script_json.segments 重建 | ✅ |
| images/*.png（31张） | 从 .agnes_source_url 侧边文件重新下载 | ✅ |
| 配音 + 字幕 | 调用 /v_factory/api/jobs/tts 重新 TTS 生成 | ✅ |
| segments/*.mp4（31个） | 从 segments_wan/ 归档目录复制恢复 | ✅ |
| 数据库路径 | 逐个字段回写 | ✅ |

## 关键命令

### 查看 job 当前状态
```sql
SELECT id, status, stage, audio_path, subtitle_path, intro_path, final_path
FROM video_job WHERE id=29;
```

### 查看 segment 路径
```sql
SELECT segment_index, image_path, clip_path, status
FROM video_segment WHERE job_id=29 ORDER BY segment_index;
```

### 从 job_log 提取 script_json
```sql
SELECT message FROM job_log WHERE id=2887;
```

### 从图片侧边文件还原
```bash
cd /path/to/media/29/images/
for f in *.agnes_source_url; do
  idx="${f%.png.agnes_source_url}"
  url=$(cat "$f" | head -1)
  curl -sS -o "${idx}.png" "$url"
done
```

## 注意

- action="script" 会清空整个脚本及下游产物（script_json、video_segment、音频、视频、图片）
- action="tts" 只清 TTS 产物（narration.mp3、clips/*.mp3、subtitles.srt）
- action="segment/*" 按 scope 清图片/视频/两者
- segments_wan/ 是重跑前自动归档目录，不会被清除操作波及
