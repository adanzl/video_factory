# 日常故事流水线

> 更新日期：2026-07-24。  
> 流水线 DB 值：`pipeline=chat`（常量名 `PIPELINE_DAILY_STORY`）。  
> 提示词索引见 [提示词构建.md](./提示词构建.md) §D。

---

## 1. 端到端

```text
主题 / 生成剧本（daily_story 表）
  → create_job（video_job, stage=script, pending）
  → script → tts → segment → intro → merge → publish
```

- 故事层：`DailyStoryMgr` + `api_daily_story`
- 视频层：`worker/stages/daily_story/*` + 共享
  `Segment` / `Intro` / `Merge` / `Publish`
- 建任务只落 `pending`，需在任务页手动跑阶段
  （无独立 cron）

## 3. 五类矛盾与独立提示词流水线

日常故事按 **矛盾类型 A–E** 各走一条写作线路，避免全部滑成
「抢东西 / 分公平」（原 C 向）。类型与提示词、质检一一对应，
不散落 `if` 例外。

### 3.1 类型一览

| 代码 | 名称 | 笑点核心 |
| --- | --- | --- |
| A | 权威翻车 | 姐姐管教/立规，被自己的话或双标反噬 |
| B | 结盟翻车 | 姐弟偷偷联手，露馅或互相甩锅 |
| C | 公平执念 | 争同一资源，两套公平规则互撞 |
| D | 字面执行 | 按叮嘱字面做，后果跑偏再回旋镖 |
| E | 妈妈破功 | 妈妈讲理/立规，被孩子闭环反问破功 |

定义与关键词、层 patterns、修订 hint：
`story_type_lines.py`（`StoryTypeLine` + `STORY_TYPE_LINES`）。

### 3.2 选题如何锁定类型

- 函数：`select_story_type_tag(theme)`（主题关键词打分）。
- 无匹配时在已校准类型 **A / C** 中随机（不再排除 D）。
- 生成时可强制：`llm_mgr.generate_daily_story(theme, story_type=…)`、
  脚本 `preview_daily_story_batch.py --story-type A`。
- 修订/质检：从 `punchline_explain` 解析类型码
  （`parse_story_type_code`），与选题不一致时以 punchline 为准。

### 3.3 提示词拼装（按阶段）

共用合同在 `prompts.py` 的 `_DAILY_STORY_SYSTEM_SHARED`
（单冲突、轮流说、好笑硬约束等）。锁定类型后叠加该类的
`StoryTypeLine` 字段：

| 阶段 | 入口 | 类型相关字段 |
| --- | --- | --- |
| D1 主题 batch | `build_daily_story_theme_prompts` | `theme_user_append` |
| D2 正文 | `build_daily_story_prompts` | `prompt_block`、锚定与收束 user |
| D2b 开场 | `build_daily_story_opening_prompts` | `opening_*_append` |
| 校验重试 | `build_daily_story_retry_user` | `retry_soft_close_hint` 等 |
| 观感修订 | `refine_daily_story_for_quality` | escalation/closing revision hint |

D1 可传 `type_code`；D2 可传 `story_type`。开场类型由 body 的
`punchline_explain` 解析。

正文生成：`llm_deepseek._generate_daily_story_body` 内
`_select_story_type(theme)` 或调用方传入的 `story_type`。
全流程：`generate_daily_story` = 正文 → 开场 → `stitch` → 校验。

**A 类当前重点（好笑）**：`本场一锤`（可画细节）→ L3–L4
**埋句**（灿灿可引用原话）→ **末四拍**
（引话→那不一样→哪里不一样→哼）。详见 `story_type_lines._LINE_A`。

B/C/D/E 已有 `prompt_block` 与层 patterns；结构与 A 对齐时
补全 `body_user_anchor`、opening/theme/retry 等字段（进行中）。

### 3.4 质检路由

- 入口：`score_daily_story` → `resolve_quality_profile(story)`。
- 配置：`quality_by_type.TYPE_QUALITY_PROFILES`（每类
  `TypeQualityProfile`：末段 scorer、妈妈戏份、软收 markers 等）。
- 层数推进：始终用 **该类型自己的** `layer_patterns`
  （`layer_patterns_for_story`，不再回退 C）。

生成侧重试：`llm_mgr` 目标 **≥85**；未达标则
`build_quality_revision_hints` + `refine_daily_story_for_quality`
（最多 3 次整稿 + 定向修订），与校验硬卡并行。

## 4. 观感评分标准

实现：`quality.py` + `quality_by_type.py`。结果写入
`story.quality`：`grade` / `score` / `summary` / `reasons`。

### 4.1 总分构成

| 部分 | 范围 | 含义 |
| --- | --- | --- |
| **结构分** | 0–**80**（上限） | 格式、类型收束形态、推进层数、节奏、硬伤扣分 |
| **好笑分** | 0–**20** | 一锤场面、收束扣原话、具体细节；硬伤压低好笑分 |
| **总分** | 0–100 | `min(100, 结构 + 好笑)`，并按下表封顶 |

#### 档位（与产品口径一致）

- **≤80**：仅结构达标或好笑不足（好笑 &lt; 5 时总分不超过 80）。
- **≥85**：好笑分 **≥ 5**（好笑达标）。
- **≥95**：好笑分 **≥ 15**（很好笑）；好笑 5–14 时总分 **≤ 94**。

`reasons` 中会附带 `结构NN`、`好笑MM` 便于排查。

### 4.2 结构分（≤80）主要项

- 基础分与跑题、`conflict_core`、发现开场、punchline 类型标签。
- 妈妈裁判/偏多台词、同人连说、甩妈/和解/耍赖/无破功软收等硬伤。
- 冲突推进层数（按类型 `layer_patterns` 分段命中）。
- 绕圈与中段复读（辩论词连击、管教口头禅复读等）。
- 类型末段 scorer（如 A 的追问闭环、C 的回旋镖等），计入结构侧。

### 4.3 好笑分（0–20）主要项

- **一锤场面**：中段可画细节（数字、算错/弹错、时长等）。
- **收束扣原话**：末段「你刚才说/明明说…」能在前文对上。
- **硬伤**：收束引话无出处、A 末四拍不完整、模板复读
  「哪里不一样」、偏 C 式空喊「不公平」等，压低或清零好笑分。

修订提示：观感 cons 含「好笑不足」「无出处」「末四拍」时，
`build_quality_revision_hints` 会追加 **【好笑】** 段，并与
`StoryTypeLine` 的 closing/escalation hint 合并。

### 4.4 等级文案

| 分数 | grade |
| --- | --- |
| ≥ 70 | 好 |
| 45–69 | 中 |
| &lt; 45 | 偏弱 |

## 5. 待做（按优先级）

### P1 — 剧本质量（单冲突）✅

已落地 `conflict_core` + 提示词【单冲突】硬约束 +
校验拦截后半跑题（学校/体育课/告爸爸等）+ 重试禁换冲突。

发现开场：正文与发现句分两步生成，拼入 `dialogue`
（`discovery_opening`），避免一上来就互怼不知争什么。

旧稿可能无 `conflict_core`；新生成必填。

### P2 — 分镜节奏验收

提示词已给目标镜数 +「转折特写不拆碎」，但无 QA。

可选：

- 轻量规则：破功 / 反驳句附近须有 `特写`
- 镜数超出目标区间则重切 / 后处理并镜
- 或 script 后处理自动并镜

### P3 — 分发钩子

- 封面（chat）：优先「特写」分镜图，其次非开场关键帧，
  再回退分镜 1；片头/开场仍跟第一镜气质（方案 C）
- 发布标题可比片内标题更钩（仍面向亲子向）

### 暂缓

| 项 | 原因 |
| --- | --- |
| 音效（摔门 / 啊？） | 已有 BGM，收益变小 |
| 建任务自动开跑 | 刻意保持手动；文案已改准 |
| `publish/done` 残留状态 | 体验问题，与观感无关 |
