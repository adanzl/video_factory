# 日常故事流水线

> 更新日期：2026-07-22。  
> 流水线 DB 值：`pipeline=chat`（常量名 `PIPELINE_DAILY_STORY`）。  
> 提示词索引见 [提示词构建.md](./提示词构建.md) §D。

---

## 1. 端到端

```text
主题 / 生成剧本（daily_story 表）
  → create_job（video_job, stage=script, pending）
  → script → tts → segment → intro → merge → publish
```

- 故事层：`DailyStoryMgr` + `api_daily_story`
- 视频层：`worker/stages/daily_story/*` + 共享
  `Segment` / `Intro` / `Merge` / `Publish`
- 建任务只落 `pending`，需在任务页手动跑阶段
  （无独立 cron）

## 2. 已落地（吸引力相关）

| 项 | 说明 |
| --- | --- |
| 剧本字数 | 正文硬卡 280–340；提示词写作目标约 390–430 |
| | （先铺够再压回硬卡）；开场另抽另验，全文不卡总字数 |

| | **走配置 thinking**；单独校验，不并入正文硬卡 |
| 开场钩子 | 正文禁止再写发现；锚点靠 conflict_core |
| 字数重试 | 按上一稿分向：偏短 `revise_expand`（只增不删）； |
| | 偏长 `revise_trim`（只删不增）；瞄准 300–320； |
| | ≥4 次；重试走配置 thinking（首稿仍硬关） |
| 关键帧 i2v | 仅 `特写` → `agnes_i2v`；开场首镜提示强制特写 |
| | 定格冲突峰值，以补回开头吸引力 |
| 生成接口 | `/generate`、`/regenerate` 异步：先落 |
| | `processing`，后台写 `active`/`failed` |
| `daily_story.status` | `processing` / `active` / `failed`； |
| | 列表自动轮询生成中条目 |
| 校验失败日志 | `ValueError` 只打一行，不甩堆栈 |
| 默认语速 | `3.6` 字/秒（`DEFAULT_DAILY_STORY_*`） |
| 句间隔 | 默认 `0.2s`，写入 |
| | `info.tts.speaker_configs.phrase_gap_sec` |
| TTS rate | 昭昭 `1.15`、灿灿 `1.30`、妈妈 `1.20` |
| 分镜提示 | 目标约 6–14 镜；开场首镜须特写；转折用特写但不拆碎 |
| 出图提示 | 强调夸张表情；蜡笔 I2I 前缀不变 |
| Ken Burns | `ken_burns_slow` zoom `1.16` |
| 成片 BGM | merge 阶段可选音频素材库曲目 + 音量 |

| 方向 | 默认横屏 `landscape`（不改竖屏） |

关键代码：

- `backend/app/services/daily_story/prompts.py`
- `backend/worker/stages/daily_story/{script,tts}.py`
- `backend/app/services/daily_story/daily_story_mgr.py`
- `frontend/.../StageMerge.vue`（BGM）
- `frontend/.../StageChatTts.vue`（音色 / rate / gap）
- `frontend/.../StageSegment.vue`（关键帧下拉）

关键帧解析：`seg.info.video_provider` → job → visual_mode →
`CLIP_PROVIDER`。日常 script **先**对 `特写` 标
`agnes_i2v`（开场吸引力靠提示强制首镜特写，不再无条件标
seg1），再 `fill_image_prompts`：关键帧写人物微动 +
固定机位（禁推近放大），其余写环境微动。`script.setting`
贯通 visual_brief / 文生图，强制开场写清客厅等地点。API：
`POST /segment/updateInfo`。

## 3. 待做（按优先级）

### P1 — 剧本质量（单冲突）✅

已落地 `conflict_core` + 提示词【单冲突】硬约束 +
校验拦截后半跑题（学校/体育课/告爸爸等）+ 重试禁换冲突。

发现开场：正文与发现句分两步生成，拼入 `dialogue`
（`discovery_opening`），避免一上来就互怼不知争什么。

旧稿可能无 `conflict_core`；新生成必填。

### P2 — 分镜节奏验收

提示词已给目标镜数 +「转折特写不拆碎」，但无 QA。

可选：

- 轻量规则：破功 / 反驳句附近须有 `特写`
- 镜数超出目标区间则重切 / 后处理并镜
- 或 script 后处理自动并镜

### P3 — 分发钩子

- 封面（chat）：优先「特写」分镜图，其次非开场关键帧，
  再回退分镜 1；片头/开场仍跟第一镜气质（方案 C）
- 发布标题可比片内标题更钩（仍面向亲子向）

### 暂缓

| 项 | 原因 |
| --- | --- |
| 音效（摔门 / 啊？） | 已有 BGM，收益变小 |
| 建任务自动开跑 | 刻意保持手动；文案已改准 |
| `publish/done` 残留状态 | 体验问题，与观感无关 |
