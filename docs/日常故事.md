# 日常故事流水线

> 更新日期：2026-07-21。  
> 流水线 DB 值：`pipeline=chat`（常量名 `PIPELINE_DAILY_STORY`）。  
> 提示词索引见 [提示词构建.md](./提示词构建.md) §D。

---

## 1. 端到端

```text
主题 / 生成剧本（daily_story 表）
  → create_job（video_job, stage=script, pending）
  → script → tts → segment → intro → merge → publish
```

- 故事层：`DailyStoryMgr` + `api_daily_story`
- 视频层：`worker/stages/daily_story/*` + 共享
  `Segment` / `Intro` / `Merge` / `Publish`
- 建任务只落 `pending`，需在任务页手动跑阶段
  （无独立 cron）

## 3. 待做（按优先级）

### P0 — 关键帧 i2v

现状：`info.video_provider` 为**任务级**，整批分镜共用
`ffmpeg` / `wan_i2v` / `agnes_i2v`。

目标：多数镜仍 `ffmpeg`；转折 / 特写等关键帧走 i2v。

必做：

1. `video_segment` 增加分镜级 `video_provider`（可空 =
   回落 job / 全局默认）
2. 出 clip 时按镜解析 provider（`media_mgr` /
   `clip_mgr`）
3. 任务页 `StageSegment` 支持单镜选择 / 批量标关键帧
4. （可选）script 后按 `shot_type` 或情绪句自动标关键帧

### P1 — 剧本质量（非字数）

字数已能打进区间（例：#19 鞋带案 28 句 / 320 字），
但仍易「后半跑题」（岔开体育课、告爸爸等）。

建议提示词硬化：

- 禁止岔开原冲突；全程围着同一件小事加码
- 每 6–8 句须有小反转，且仍服务同一冲突
- 妈妈只收口 / 点破，不新开账

### P2 — 分镜节奏验收

提示词已要求转折句单独成镜，但无 QA。

可选：

- 轻量规则：破功 / 反驳句附近须有 `特写`
- 或 script 后处理自动拆镜

### P3 — 分发钩子

- 封面定「冲突瞬间」帧，而非平铺群像
- 发布标题可比片内标题更钩（仍面向亲子向）

### 暂缓

| 项 | 原因 |
| --- | --- |
| 音效（摔门 / 啊？） | 已有 BGM，收益变小 |
| 建任务自动开跑 | 刻意保持手动；文案已改准 |
| `daily_story.status` 状态机 | 现恒 `active`，非成片瓶颈 |
| `publish/done` 残留状态 | 体验问题，与观感无关 |
